What is Apache Hadoop Ozone?

Ozone is a new subproject of Apache Hadoop. It provides an object store semantic (something like s3) for Hadoop.

It's based on HDDS (aka Hadoop Distributed Data Storage) which is an other new for Apache Hadoop. It provides a storage layer for other higher level componets (such as Ozone)

Scalable

Ozone is designed to scale to tens of billions of files and blocks and, in the future, to every larger number of smaller objects.

Small files or huge number of datanodes and block reports are no longer problems.

Consistent

Using of RAFT protocol has allowed the storage layer to be self-consistent.

Cloud-native

Hadoop Ozone works well in containerized environment.

Quadra is a new component based on the HDDS storage layer which could provide provide raw device access to Kubernetes or any other CNI compatible cloud solutions.

Question and Answers

Where can I download it?

Apache Hadoop Ozone is not yet released. It's developed on the trunk branch of Apache Hadoop. And short-term will be released independently from the main Hadoop release.

How can I contribute

The contribution process is the same with any other Hadoop subproject.

Does it mean a new Hadoop server component?

The worker part of Ozone/HDDS can be started as a HDFS Datanode plugin. Both Ozone and HDDS requires one additional master component (Ozone Manager and Storage Container Manger)

How can I try it out?

There is a katacoda which could be used. Please note that it contains a demonstration verison which is not released by Apache.

For the Apache version you can download and build the latest versoin from trunk
